<Note title="What is WiFiPumpkin3 Pro?">
A professional-grade framework for adversarial emulation, Wi-Fi security testing, and rogue access point deployment built for red teamers and security professionals.

### Welcome

Welcome to the documentation for **WiFiPumpkin3 Pro** — a modern, extensible platform for rogue access points, credential harvesting, and transparent data interception.

![wifipumpkin3-pro running](/images/pro/installation/wifipumpkin3-pro-kalinux2015-2.png)

### WebUI

WiFiPumpkin3 Pro comes with a sleek, built-in **Web User Interface** designed for ease of use and remote operation.  
It provides full control over access point modes, plugins, logs, and MITM proxies — all from your browser.

<Tip title="Cross‑Platform & Headless Friendly">
The WebUI allows you to manage rogue AP deployments even on remote devices like Raspberry Pi or headless VMs.
</Tip>

#### Login Screen

To access the interface, simply navigate to `http://<your-ip>:8000` after launching the service.  
You’ll be prompted to log in using your admin credentials.

#### Dashboard Overview

The dashboard provides a centralized view of all core modules and their status — including RogueAP mode, connected clients, DNS/DHCP status, and running modules.

![Dashboard Screenshot](/images/webui/dashboard.png)

#### Features of the WebUI

- ✅ Start/Stop **RogueAP Modes** with a single click  
- 📡 Monitor connected clients in real-time  
- 🛠️ Manage **modules** dynamically  
- 🌐 Deploy phishing pages via captive portal templates  
- 🔎 View captured credentials, DNS queries, and HTTP logs

### New in `v3.0.0-beta1`

This beta expands WP3 Pro beyond core AP operations with stronger automation, mobile workflows, and secure remote control.

#### Platform and UI Enhancements

- Full mobile support with PWA app experience
- Interactive Shell UI in dashboard for operational diagnostics
- `Find My Devices` for local WP3 Pro endpoint discovery
- `Recent Devices` for fast reconnect workflows
- Native HTTPS/TLS for `web.ui` and `api.rest` with auto-TLS setup flow

#### Automation and Session Continuity

- Session `var` support in WP3 Pulp automation
- Safer command runtime with `exec run`, `exec capture`, and `exec allowed`
- Persistent session lifecycle management with `session.manager`
- Script catalog and validation pipeline with `scripts.runner`

#### Field and Recon Readiness

- Geolocated Wi-Fi collection with `wardriving`
- Local network profiling and port scanning with `lan.recon`
- Runtime battery awareness with `battery.monitor`
- Alternative control channel support via Bluetooth PAN (`bt.pan`)
- Faster phishing simulation setup with PhishPortal template download workflows
